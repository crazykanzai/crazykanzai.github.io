AI agent入门——学习mini-swe-agent（1）

鼠鼠在ai方面真的是0基础，学姐推荐了github上的mini-swe-agent项目，这篇文章零散地记录一下学习过程。

一、python环境配置

1、创建虚拟环境

为了不污染其他项目的环境，在开发之前要先创建一个虚拟环境

# 创建虚拟环境
python -m venv venv

# 激活环境 (Windows)
.\venv\Scripts\activate

# 激活环境 (Mac/Linux)
source venv/bin/activate

每一个项目有独立的环境，避免包版本引起的冲突。

2、可编辑模式

pip install -e .

注意！最后有一个“.”

-e (Editable): 代表“可编辑模式”。这意味着你修改了本地的代码后，不需要重新安装，改动会立刻生效。

. (Dot): 代表“当前目录”。它告诉 pip：“就在当前文件夹里找安装配置文件“

二、代码研读

我现在手上没有API，所以先研究一下静态代码

1、hello_world.py

这段代码是一个 CLI（命令行界面）程序的入口。它的作用是：定义一个命令，让用户可以通过命令行输入任务（Task）和模型（Model），然后启动一个 Agent 去执行。

模块导入

import typer	#用于创建命令行工具
import yaml		#加载配置文件

from minisweagent import package_dir	#导入核心组件
from minisweagent.agents.default importDefaultAgent		#默认的代理类
from minisweagent.environments.local import LocalEnvironment	#本地环境类，用于执行命令
from minisweagent.models.litellm_model import LitellmModel		#语言模型的封装类

使用typer创建一个命令行应用

app = typer.Typer()

定义主函数

def main(
    task: str = typer.Option(..., "-t", "--task", help="Task/problem statement", show_default=False, prompt=True),
    model_name: str = typer.Option(
        os.getenv("MSWEA_MODEL_NAME"),
        "-m",
        "--model",
        help="Model name (defaults to MSWEA_MODEL_NAME env var)",
        prompt="What model do you want to use?",
    ),
) -> DefaultAgent:

特意声明了变量类型。在命令行里-t或者–task后面的字符串会被赋值给task变量。 prompt=True表示如果用户没有输入参数就会一直等着，类似scanf()。

函数体

logging.basicConfig(level=logging.DEBUG) #开启这个设置所有调试信息都会被打印出来
#这一段代码是核心
agent = DefaultAgent(
    LitellmModel(model_name=model_name),	
    LocalEnvironment(),					
    **yaml.safe_load(Path(package_dir / "config" / "default.yaml").read_text())["agent"],
)

LitellmModel()初始化”大脑“，LocalEnvironment()初始化环境，让agent能操作本地文件

**yaml：双星号解包，它读取了一个yaml文件，把它转化成字典，再用__**__把它变成一堆关键字数据传给DefaultAgent。

agent.run(task)
return agent

启动Input -> [思考 -> 行动 -> 观察结果 -> 再思考 -> 再行动…] -> Output的循环

程序入口检查

if __name__ == "__main__":	#如果我是被直接运行的（不是被别的脚本调用的），那就执行下面的代码
    app()	#启动 Typer 应用，开始处理命令行参数并调用上面的 main 函数

相当于c语言的‘int main(){}

总结：这段代码的执行流程

如果在命令行输入：

python your_script.py --task "帮我写个贪吃蛇" --model "gpt-4"

Python 解释器启动，从 if __name__ == "__main__": 开始执行。

app() 捕获到命令行参数，发现用户想执行 main 命令。

app() 自动解析：

  • 把 "帮我写个贪吃蛇" 赋值给 task。
  • 把 "gpt-4" 赋值给 model_name。

进入 main 函数：

  • 读取 YAML 配置文件。
  • 组装一个 DefaultAgent（装上 GPT-4 大脑，连上本地环境）。

执行 agent.run(task)，Agent 开始干活。

2、default.py

这个脚本是代理的核心逻辑

定义了DefaultAgent类

class DefaultAgent:
    def __init__(self, model: Model, env: Environment, ...):
        # ...初始化代码...

self 指针：在 Python 的类方法中，第一个参数永远是 self。这完全等同于 C++ 里的 this 指针

__init__：这是构造函数。当你写 agent = DefaultAgent(...) 时，Python 自动调用这个函数来初始化结构体里的变量（比如设置 self.cost = 0）

核心循环：run 函数

这是整个agent最关键的部分

def run(self, task: str = "", **kwargs) -> dict:
    """Run step() until agent is finished. Returns dictionary with exit_status, submission keys."""
    self.extra_template_vars |= {"task": task, **kwargs}
    self.messages = []
    self.add_messages(
        self.model.format_message(role="system", content=self._render_template(self.config.system_template)),
        self.model.format_message(role="user", content=self._render_template(self.config.instance_template)),
    )
    while True:
        try:
            self.step()		#核心的一步，思考并行动。不断地调用这一步，直到收到退出信号
            
        except InterruptAgentFlow as e:	#错误捕获
            self.add_messages(*e.messages)
        except Exception as e:
            self.handle_uncaught_exception(e)
            raise
        finally:
            self.save(self.config.output_path)
        if self.messages[-1].get("role") == "exit":
            break		#退出条件：如果最后一条消息是"exit"就退出循环
    return self.messages[-1].get("extra", {})

step函数

def step(self) -> list[dict]:
        """Query the LM, execute actions."""
        # 先 query() 询问大脑，拿到指令
        # 然后 execute_actions() 执行指令
        return self.execute_actions(self.query())

query函数：思考环节

def query(self) -> dict:
        # 1. 检查限制 (如同看门狗 Watchdog)
        if 0 < self.config.step_limit <= self.n_calls ...:
            raise LimitsExceeded(...) # 如果步数超了，抛出异常停止
            
        self.n_calls += 1
        
        # 2. 调用大模型 (LLM)
        # 把所有的聊天记录 (self.messages) 发给大脑，问它：“下一步该干嘛？”
        message = self.model.query(self.messages)
        
        # 3. 记录成本 (Cost)
        self.cost += ...
        
        # 4. 更新记忆
        self.add_messages(message)
        
        return message

这里的 self.messages 就是 记忆 (Memory)。 每次调用模型前，它都会检查“我是不是太累了？”（Step limit）或者“我是不是花太多钱了？”（Cost limit）。如果一切正常，它就把最新的对话记录发给 LLM。

execute_actions函数：行动环节

def execute_actions(self, message: dict) -> list[dict]:
        # List Comprehension (Python 特有的列表推导式)
        # 翻译成 MATLAB：
        # for i = 1:length(actions)
        #    outputs(i) = env.execute(actions(i));
        # end
        outputs = [self.env.execute(action) for action in message.get("extra", {}).get("actions", [])]
        
        # 把执行结果 (Observation) 格式化后，再次存入记忆
        return self.add_messages(*self.model.format_observation_messages(...))

这里有一个很 Pythonic 的写法：[... for ... in ...]。 它的逻辑是：

  1. 从 LLM 的回复 (message) 里提取出 actions（比如 ['run_test.py', 'ls -l']）。
  2. 遍历每一个动作，调用 self.env.execute(action) 去真的执行它（这就是“手”在动）。
  3. 拿到结果 outputs（比如终端的输出文字）。
  4. 把这些结果打包成新的消息，塞回 self.messages 列表里。

serialize函数

将代理的状态序列化为 JSON 格式，包含配置信息、消息历史等。

save函数

将代理的执行轨迹保存到文件中。

总结：

DefaultAgent是整个项目的核心类，负责：

  1. 管理代理的配置（通过 AgentConfig）。
  2. 与语言模型交互（query() 方法）。
  3. 执行模型返回的操作（execute_actions() 方法）。
  4. 记录和保存执行轨迹（serialize()和 save()方法）。
